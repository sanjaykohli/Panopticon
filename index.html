Technical Overview

Panopticon demonstrates client-side data collection techniques. It illustrates how modern web applications can identify and track users without cookies, relying instead on browser APIs, hardware differences, and network behaviors.

For Developers

Understand the code behind these diagnostics. View the source to see how Canvas Fingerprinting, WebRTC IP leakage, and Audio Context analysis are implemented in pure JavaScript.

Canvas Fingerprinting
Critical Risk
Technique

The browser renders hidden text and gradients. Subtle differences in GPU rendering, anti-aliasing, and OS font handling produce unique pixel data. We hash this data with SHA-256.

Privacy Impact

Creates a persistent identifier that remains consistent even after clearing cookies or using private browsing. Incognito mode does NOT protect you.

WebRTC IP Leak
Critical Risk
Technique

WebRTC establishes peer-to-peer connections. To facilitate this, it may expose local LAN IP addresses and, in some configurations, the real public IP behind a VPN.

Privacy Impact

Reveals internal network architecture and potentially bypasses VPN protections to disclose physical location.

Hardware & Audio Context
Medium Risk
Technique

Scripts analyze CPU core count, RAM estimates, battery behavior, and audio signal processing characteristics unique to each device's hardware stack.

Privacy Impact

Contributes to a composite fingerprint. While individual metrics are not unique, the aggregate profile is highly specific to you.
